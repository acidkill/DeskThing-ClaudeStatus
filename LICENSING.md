## What changed from upstream Clawdmeter

The upstream [Clawdmeter](https://github.com/HermannBjorgvin/Clawdmeter) repository bundles assets whose redistribution rights are unclear â€” Anthropic-proprietary fonts (Tiempos Text, Styrene B) and Clawd mascot pixel-art scraped from a fan site. The upstream author flags this explicitly. **This port handles those assets deliberately:**

- **Code**: every line under `server/`, `src/`, `shared/`, `tests/`, and `scripts/` is original work by AI-Flow NOWAK and contributors, redistributable under Apache-2.0.
- **Mascot art**: this fork bundles the original Clawd sprite JSONs (13 upstream animations) plus two new bespoke idle animations (`idle_strawberry`, `idle_bubbles`) generated by `scripts/generate-idle-sprites.mjs`. The two new sprites are derivative work (they reuse the upstream Clawd body pose) and inherit the same restricted distribution terms â€” they are not separately redistributable.
- **`MeterMascot.tsx`** (the original SVG "Pip" mascot) remains in the codebase as an unused fallback. It is hand-authored, Apache-2.0, and may be reactivated by reverting the screen-component changes if the Clawd sprites need to be removed for a public release.
- **Fonts**: no proprietary fonts ship in this repo. Tailwind tokens reference `Inter` and `JetBrains Mono` family names with a graceful fallback to `system-ui` / `ui-monospace`. Self-hosting open-licence Inter + JetBrains Mono WOFF2 under `public/fonts/` is a planned follow-up.
- **Anthropic branding**: this is a third-party tool that reads the Anthropic API's rate-limit response headers. No Anthropic wordmark, logo, or brand fonts appear anywhere in the UI text or chrome. The Clawd mascot appears because that is the visual identity inherited from upstream Clawdmeter, not because the project claims affiliation with Anthropic.

## Public-fork instructions

If you want to publish a public mirror of this code:

1. Delete `assets-proprietary/` from the working copy.
2. Add `assets-proprietary/` back to `.gitignore`.
3. Revert the v0.3.1 screen-component changes (`UsageScreen`, `SplashScreen`, `SettingsScreen`) so they fall back to `<MeterMascot />`.
4. Regenerate `deskthing/icons/claude-status.svg` from a non-Clawd source.
5. Rebuild â€” `hasClawdSprites()` returns `false`, the UI falls back to the original Pip mascot, and the public build ships with only Apache-2.0 assets.

The build pipeline is intentionally structured so the Clawd dependency is contained in `assets-proprietary/` + three screen components; everything else stays public-safe.
